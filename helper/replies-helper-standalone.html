How this works

Offline mode never talks to Ghost. It just builds the properly-formatted <script type="application/json" id="social-replies">...</script> block from your URL list. You then paste it into Ghost admin yourself (post → gear icon → Code injection → Post footer). To preserve URLs already on a post, paste the current script block into the "Current footer" field first — the helper merges and de-duplicates.

Direct mode generates a short-lived JWT in your browser (HS256, 5-minute expiration, audience /admin/) using the Web Crypto API, then calls Ghost's Admin API directly. No PHP proxy needed. Two caveats:

  1. Your Admin API key is stored in localStorage after you click Connect. Anyone with access to this browser can read it. Use "Forget saved key" when you're done if you're worried.
  2. Ghost may reject browser-origin requests with a CORS error. If that happens, fall back to Offline mode — it always works.
